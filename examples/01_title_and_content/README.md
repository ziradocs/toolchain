# Title and Content Examples - Flex Mode

This folder contains modern, realistic examples demonstrating all of SlideLang's capabilities for handling titles, content, and basic presentation elements, organized by specialty and use case.

## 🎯 Main Files (Realistic Use Cases)

### `content_platform_launch_flex.slidelang`
**Primary use case** - Content platform launch:
- Complete product launch presentation
- Coherent business narrative
- All the DSL's basic elements integrated
- Advanced use of variables and specialized layouts

### `01_title_and_content_flex.slidelang`
**Technical reference example** covering all core functionality:
- Variables and dynamic configuration
- Advanced text formatting
- Structured lists and tables
- Multimedia and images

### `01.1_frontmatter_variables_flex.slidelang`

## 🚀 Philosophy Behind the Updated Examples

### Realistic Use Cases
- **Coherent narrative**: Each file tells a complete business story
- **Realistic data**: Variables, metrics, and content based on real scenarios
- **Direct applicability**: Examples that can be adapted to real projects

### Specialized Organization
- **One concept per file**: Each specialized file focuses on specific elements
- **Logical progression**: From basic to advanced
- **Quick reference**: Easy to find examples of specific elements

### Best Practices
- **Specialized layouts**: Using appropriate layouts for each content type
- **Dynamic variables**: Reusable, maintainable configuration
- **Clear structure**: Well-organized frontmatter and logical sections

## 📖 How to Use These Examples

1. **To learn**: Start with `content_platform_launch_flex.slidelang`
2. **For reference**: Check the specialized files by element
3. **For development**: Use the variables and structures as templates
4. **For migration**: Compare flex vs. strict versions


1. **Dynamic variables** for reusable content
2. **Inline documentation** explaining each element
3. **Logical progression** from simple to advanced concepts
4. **Realistic use cases** instead of dummy data
5. **Complete coverage** of all functionality
6. **Runnable examples** that work as reference
